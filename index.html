1 Steam Tags Analysis

시장 최대 규모의 Digital Distribution Platform인 Steam에 등록된 모든 게임의 태그와 리뷰 정보를 긁어와 이를 통해 게임을 장르화하고 장르와 리뷰 간 상관관계를 통계적으로 분석해보았습니다. 데이터는 R 3.5.1의 rvest 0.3.2 패키지를 이용하여 Steam.com에서 직접 긁어왔으며, XML 3.98 패키지로 데이터를 가다듬어 .csv로 저장하였습니다. 이후 eigenvector centrality, pagerank 등 간단한 통계적 지표는 Gephi 0.9.2를 사용하여 계산하였으며, Gephi 0.9.2와 igraph 1.2.2로 네트워크를 시각화하고 alluvial diagram을 만들어 보았습니다.

2 Steam Review Prediction Model

SteamSpy API를 통해 스팀에 등록된 모든 게임의 정보를 .json으로 구한 뒤, 해당 데이터를 정리하여 .csv로 저장하고 Tensorflow의 ConvNet, max pooling, batch normalization을 수차례 거쳐 RELU로 리뷰를 예측합니다. Python 3.6.7로 작업하였으며, Numpy, Pandas, Tensorflow를 사용하였습니다. Tensorflow는 Windows 10 GTX1080 환경에서 실행하였고, Tensorflow 1.12.0에 Nvidia CUDA 10.0.130과 Nvidia cuDNN 7.4.1, Microsoft Visual C++ 2015 Update 3를 이용했습니다.

3 PlayDB Crawler

국내 최고의 공연 데이터베이스 사이트인 playdb.com에 등록된 콘서트, 뮤지컬, 페스티벌 등의 공연 이름, 시작날짜, 종료날짜, 공연자, 평점 등의 정보를 긁어와 .csv로 출력 및 저장을 지원하는 R 스크립트를 만들어보았습니다. R 3.5.1로 rvest 0.3.2 패키지를 통해 크롤러를 작성하였고, XML 3.98과 plyr 1.8.4로 데이터를 정리하였습니다.

4 An Interactive Network of KAIST Professors

2015년 기준 KAIST의 모든 교수의 학부-석사-박사 진학 과정에서 나타나는 전공 변경의 시각화입니다. 원본 데이터는 KAIST 교무팀에 컨택하여 입수하였습니다. 이후 R로 데이터를 정규화하고 용도에 맞게 개선하였습니다. R 3.5.1에서 중복되는 열, 비어있는 열을 삭제한 뒤 띄어쓰기나 대소문자 등 형식이 다른 과명칭을 통일시켰습니다. 그 뒤 개인정보를 비식별화하였으며, 그 뒤에 다시 발생하는 중복열은 통합하여 edge weight 행에 가산하였습니다. 이렇게 기본적인 정규화를 진행한 뒤 Gephi 0.9.2에서 SigmaExporter 0.9.0 플러그인을 사용하여 HTML5 인터렉티브 네트워크로 내보냈습니다. 이후 가독성을 높이고 전체적으로 더 나은 UX를 제공하기 위해 index.html과 style.css, config.json를 가볍게 수정하였습니다.

5 Handwritten Digits Recognizer

사람이 손으로 쓴 0부터 9까지의 숫자의 이미지의 각 픽셀값을 한 열로 가지는 대용량 행렬의 일부를 train data로, 나머지를 test data로 정한 뒤, train data를 .csv로 읽어들여 학습하고 test data를 입력 받으면 어떤 숫자인지 출력하는 딥러닝 모델입니다. Python3로 작성되었으며, MNIST 데이터베이스에서 train data와 test data를 받아왔습니다. Keras의 ImageDataGenerator로 부족한 데이터를 보강하였으며, 이후 데이터를 normalize하였습니다. 준비가 끝난 데이터는 keras의 convolutional network, max pooling과 batch normalization 과정을 수차례 거친 뒤 RELU와 Softmax activation 함수를 통해 결론에 도달합니다. Evaluation 결과 인식 정확도는 99.9%였습니다.

6 Twitter Word Cloud

트위터 해시태그로 워드 클라우드를 만드는 R 스크립트입니다. 원하는 해시태그를 검색하면 해당 해시태그를 가진 트윗 N개를 긁어와 발생 빈도에 따라 크기를 결정한 뒤 워드 클라우드 형태로 출력합니다. R.3.5.1을 사용하였으며, rtweet 패키지를 이용해 트위터 개발자 계정에 할당되는 API로 트위터를 매우 쉽게 긁어올 수 있었습니다. 역시 tidytext와 dplyr, stringr 등의 패키지를 사용해 rtweet이 긁어온 데이터를 용도에 알맞는 형식으로 변환하였고, wordcloud2 패키지를 이용해 워드 클라우드를 생성하였습니다.
